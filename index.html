Document Structure

Every HTML document must follow a specific structure. Think of it like a letter - it needs an envelope (DOCTYPE), a header with info (head), and the actual message (body).

HTML 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <!-- All visible content goes here -->
    <h1>Hello World</h1>
    <p>This is a paragraph.</p>
  </body>
</html>
Key Points:
  • <!DOCTYPE html> must be the first line - tells the browser to use modern rendering (standards mode)
  • The <head> contains information ABOUT the page, not visible content
  • Only elements inside <body> appear on the screen
  • Always include the viewport meta tag for mobile responsiveness!

The DOCTYPE Declaration

The <!DOCTYPE html> declaration is required at the very beginning of your HTML document. It tells the browser which version of HTML you're using. For HTML5 (the current standard), it's simply <!DOCTYPE html>.

Important: Without DOCTYPE, browsers enter "quirks mode" and render pages inconsistently. Always include it!

Essential Meta Tags

Meta TagPurposeRequired?

Common Mistakes:
  • Forgetting the viewport meta tag (breaks mobile layout!)
  • Using wrong character encoding (causes weird symbols)
  • Putting meta tags in the body (they belong in head)
  • Missing closing tags for html, head, or body
Headings

Headings create a document outline. HTML has six levels of headings, from h1 (most important) to h6 (least important).

<h1>Main Page Title (ONE per page!)</h1>
<h2>Major Section</h2>
<h3>Subsection</h3>
<h4>Sub-subsection</h4>
<h5>Minor heading</h5>
<h6>Smallest heading</h6>
Only ONE h1 per page! The h1 should describe the main topic of the entire page. Using multiple h1s confuses search engines and screen readers.
Paragraphs and Text Formatting
<p>This is a paragraph of text.</p>

<p>This is <strong>important</strong> text (bold, semantic).</p>

<p>This is <em>emphasized</em> text (italic, semantic).</p>

<p>This text has <br>a line break in the middle.</p>
Semantic vs Presentational:
  • <strong> = important content (bold)
  • <em> = emphasized content (italic)
  • These carry meaning, not just visual style
  • Use instead of <b> and <i> which are purely visual

Links (Anchor Elements)

The href Attribute

The href (hypertext reference) attribute tells the browser where the link goes:

<!-- Without href - just text -->
<p>This is just <a>text</a>, not a link.</p>

<!-- With href - clickable link -->
<p>Visit <a href="https://example.com">Example.com</a>.</p>

Types of Links

1. Absolute Links - Link to other websites (full URL):

<a href="https://developer.mozilla.org">MDN Web Docs</a>

2. Relative Links - Link to files relative to current file:

<!-- Same folder -->
<a href="about.html">About Page</a>

<!-- Subfolder -->
<a href="products/item.html">Product</a>

<!-- Parent folder (go up one level) -->
<a href="../index.html">Home</a>

3. Root-Relative Links - Relative to website root:

<!-- Starts from domain root -->
<a href="/">Home Page</a>
<a href="/about.html">About</a>

Opening Links in New Tab

<a href="https://example.com" target="_blank">
  Open in New Tab
</a>
Naming Convention: NEVER use spaces in filenames! Use hyphens instead:
  • ❌ Bad: my page.html (becomes my%20page.html)
  • ✅ Good: my-page.html

CSS Link States

Links have different states that can be styled. Order matters!

/* LVHA order - very important! */

/* Default state */

  color: blue;
  text-decoration: none;
}

/* Already visited */
a:visited {
  color: purple;
}

/* Hovering over link */
a:hover {
  color: darkblue;
  text-decoration: underline;
}

/* Link being clicked */
a:active {
  color: red;
}
LVHA Order: Must be in this order:
  1. :link
  2. :visited
  3. :hover
  4. :active
If you put :hover before :link, the hover styles won't work properly!

Padding

.box {
  padding: 20px;
}

/* Top/bottom, left/right */
.box {
  padding: 10px 20px;
}

/* Top, right, bottom, left (clockwise) */
.box {
  padding: 10px 20px 15px 25px;
}

Margin

Margins work the same way as padding, but they create space OUTSIDE the element.

/* Center an element horizontally */
.container {
  margin-left: auto;
  margin-right: auto;
  /* OR shorthand: */
  margin: 0 auto;
}

Margin Collapse

What is Margin Collapse?

When two block elements' vertical margins touch, they don't add up - they collapse to the larger of the two margins.

.top {
  margin-bottom: 20px;
}
.bottom {
  margin-top: 30px;
}

/* Result: 30px gap (not 50px!) */

Important: Only vertical margins collapse. Horizontal margins always add up.

Border

/* Shorthand */
.box {
  border: 2px solid #333;
  /* border-width border-style border-color */
}

/* Individual properties */
.box {
  border-width: 2px;
  border-style: solid;  /* solid, dashed, dotted, double */
  border-color: #333;
  border-radius: 8px;   /* rounded corners */
}

/* Individual sides */
.box {
  border-top: 1px solid #ccc;
  border-bottom: 3px solid #000;
}

Box-Sizing

/* Without border-box (default) */
.box {
  width: 300px;
  padding: 20px;
  border: 2px solid black;
  /* Actual width: 300 + 20 + 20 + 2 + 2 = 344px! */
}

/* With border-box */
.box {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 2px solid black;
  /* Actual width: 300px (padding and border inside!) */
}
Best Practice - Reset Border Box:

Always include this at the top of your CSS:

* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

This makes width/height include padding and border, which is much more intuitive!

CSS Positioning

The position property changes how elements are placed in the document.

PositionBehaviorUse Case
staticNormal document flow (default)Most content
relativeOffset from normal positionSmall adjustments, positioning context
absolutePositioned relative to nearest positioned ancestorOverlays, dropdowns, tooltips
fixedPositioned relative to viewportSticky headers, floating buttons

Static (Default)

Elements with position: static follow the normal document flow. You cannot use top, left, right, or bottom with static positioning.

Relative Positioning

.box {
  position: relative;
  top: 20px;    /* Move down 20px from normal position */
  left: 30px;   /* Move right 30px from normal position */
}

/* The element still takes up its original space!
   Other elements don't move to fill the gap */
Why use relative positioning?
  • To make small adjustments without affecting layout
  • To create a positioning context for absolute children
  • To layer elements with z-index

Absolute Positioning

/* Parent creates positioning context */
.container {
  position: relative;
}

/* Child positioned relative to parent */
.overlay {
  position: absolute;
  top: 10px;
  right: 10px;
  /* Positioned 10px from top-right of .container */
}

/* Another example - centering */
.centered {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  /* Perfect center! */
}
Important: Absolute positioned elements are removed from document flow. They don't affect other elements' positions!

Fixed Positioning

.sticky-header {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  height: 60px;
  background: white;
  z-index: 100;
}

/* Stays at top even when scrolling! */

Z-Index (Stacking Order)

Controls which element appears on top when they overlap.

.back {
  position: absolute;
  z-index: 1;
}

.front {
  position: absolute;
  z-index: 2;  /* Higher = on top */
}

/* Only works on positioned elements (not static!) */
Common Mistakes:
  • Trying to use z-index on static elements (doesn't work!)
  • Forgetting to set position: relative on parent when using absolute children
  • Not accounting for space removed from flow with absolute/fixed
Flexbox

Flexbox is a powerful layout system for one-dimensional layouts (row OR column).

Setting Up Flexbox

/* The parent becomes a flex container */
.container {
  display: flex;
}

/* All direct children become flex items */
/* HTML:
   

     
Item 1
  ← flex item
     
Item 2
  ← flex item
     
Item 3
  ← flex item
   

*/

Main Properties on Container

flex-direction

Controls the main axis (direction items are laid out):

.container {
  display: flex;
  flex-direction: row;           /* Default: left to right */
  flex-direction: row-reverse;   /* Right to left */
  flex-direction: column;        /* Top to bottom */
  flex-direction: column-reverse;/* Bottom to top */
}

justify-content (Main Axis Alignment)

.container {
  display: flex;
  
  /* Distribute space */
  justify-content: flex-start;   /* Default: items to start */
  justify-content: flex-end;     /* Items to end */
  justify-content: center;       /* Items centered */
  justify-content: space-between;/* Space between items */
  justify-content: space-around; /* Space around items */
  justify-content: space-evenly; /* Equal space everywhere */
}

align-items (Cross Axis Alignment)

.container {
  display: flex;
  
  /* Vertical alignment (if row direction) */
  align-items: stretch;     /* Default: stretch to fill */
  align-items: flex-start;  /* Align to top */
  align-items: flex-end;    /* Align to bottom */
  align-items: center;      /* Center vertically */
  align-items: baseline;    /* Align text baselines */
}

Common Patterns

Center a div both horizontally and vertically:

.center-container {
  display: flex;
  justify-content: center;  /* Horizontal center */
  align-items: center;      /* Vertical center */
  min-height: 100vh;        /* Full viewport height */
}

Navigation bar:

.navbar {
  display: flex;
  justify-content: space-between;  /* Logo left, menu right */
  align-items: center;             /* Vertically centered */
  padding: 20px;
}

Flex Wrap

By default, flex items try to fit on one line. Use flex-wrap to allow wrapping:

.container {
  display: flex;
  flex-wrap: wrap;        /* Allow items to wrap */
  gap: 16px;              /* Space between items (modern!) */
}

.item {
  flex: 1 1 300px;        /* Grow, shrink, base size */
  /* OR */
  flex-basis: 300px;      /* Ideal starting width */
}

Individual Flex Item Properties

PropertyWhat it does
flex-growHow much item grows relative to others (0 = don't grow)
flex-shrinkHow much item shrinks relative to others
flex-basisStarting size before growing/shrinking
align-selfOverride align-items for this item only
/* Shorthand: flex: grow shrink basis */
.item {
  flex: 0 0 200px;  /* Fixed 200px width, no grow/shrink */
}

.item-grow {
  flex: 1;          /* Grow to fill space equally */
}

.item-double {
  flex: 2;          /* Grows twice as much */
}
Flexbox Tips:
  • Use gap instead of margins for spacing (cleaner!)
  • flex: 1 makes items share space equally
  • margin-left: auto pushes an item to the right
  • Flexbox is one-dimensional - use Grid for two-dimensional layouts

Increment/Decrement: Prefix vs Postfix

let a = 5;
let b = 5;

// Postfix: returns value, THEN increments
let x = a++;      // x = 5, a = 6

// Prefix: increments, THEN returns value
let y = ++b;      // y = 6, b = 6

// In standalone statements, no difference:
a++;   // same as ++a
Best Practice: If you don't need the return value, use i++ or i-- (more common). If you need the incremented value immediately, use ++i.

String Comparison

Strings are compared letter by letter using Unicode values (dictionary/lexicographical order):

// Lexicographical (dictionary) order
"Z" > "A";           // true
"Glow" > "Glee";     // true (o > e at position 3)
"Bee" > "Be";        // true (longer string is greater)

// Case matters!
"a" > "A";           // true (lowercase has higher Unicode value)
"Apple" === "apple"; // false

Comparing Different Types

When comparing values of different types, JavaScript converts them to numbers:

// Loose equality converts to numbers
"2" > 1;             // true ("2" becomes 2)
"01" == 1;           // true ("01" becomes 1)
true == 1;           // true (true becomes 1)
false == 0;          // true (false becomes 0)

// But strict equality does NOT convert
"2" === 2;           // false (string vs number)
true === 1;          // false (boolean vs number)

The Null and Undefined Traps

// Strict equality
null === undefined;  // false (different types)

// Loose equality - special case!
null == undefined;   // true (they are "loosely equal")

// BUT comparisons with null/undefined
null > 0;            // false (null becomes 0)
null == 0;           // false (special case!)
null >= 0;           // true (null becomes 0)

// undefined comparisons
undefined > 0;       // false (undefined becomes NaN)
undefined == 0;      // false
undefined < 0;       // false
Confusing but True:

null >= 0 is true but null > 0 and null == 0 are both false.

This happens because:

  • Comparisons convert null to 0, so null >= 0 works
  • But equality check has special rule: null only equals undefined

How to Avoid Comparison Issues:
  • Always use === and !== instead of == and !=
  • Treat comparisons with null/undefined separately
  • Use explicit conversions when unsure

Implicit (Automatic) Conversion

JavaScript automatically converts types in certain situations:

if (1) { }           // true

num.toString();      // "123"

// Alternative (less common)
num + "";            // "123" (but uses concatenation)

To Number

// Using Number()
Number("123");       // 123
Number("123.45");    // 123.45
Number("");          // 0
Number("   ");       // 0
Number("hello");     // NaN (Not a Number)
Number(true);        // 1
Number(false);       // 0
Number(null);        // 0
Number(undefined);   // NaN

// Using unary plus (shorthand)
+"123";              // 123
+true;               // 1
+"";                 // 0

To Boolean

// Falsy values (convert to false)
Boolean(0);          // false
Boolean("");         // false
Boolean(null);       // false
Boolean(undefined);  // false
Boolean(NaN);        // false
Boolean(false);      // false

// Truthy values (convert to true) - everything else!
Boolean(1);          // true
Boolean("hello");    // true
Boolean(" ");        // true (space is not empty!)
Boolean([]);         // true (empty array)
Boolean({});         // true (empty object)
Falsy vs Truthy:

There are only 6 falsy values: 0, "", null, undefined, NaN, false. Everything else is truthy!

parseInt and parseFloat

// parseInt - extracts integer from start of string
parseInt("100px");   // 100
parseInt("12.5");    // 12 (drops decimal)
parseInt("px100");   // NaN (doesn't start with number)

// parseFloat - extracts decimal number
parseFloat("12.5");  // 12.5
parseFloat("100.5px"); // 100.5

// Second parameter = radix (number base)
parseInt("0xFF", 16);  // 255 (hexadecimal)
parseInt("10", 2);     // 2 (binary)

Required HTML Structure

Every HTML file must have this exact structure:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Homepage</title>  <!-- Or "Oranges", "Strawberries" -->
  
  <!-- Reset CSS - MUST LOAD FIRST -->
  <link rel="stylesheet" 
        href="https://cdnjs.cloudflare.com/ajax/libs/meyer-reset/2.0/reset.min.css">
  
  <!-- Shared styles - LOAD SECOND -->
  <link rel="stylesheet" href="styles/A1.css">
  
  <!-- Page-specific styles - LOAD LAST -->

</head>
<body>
  <div id="header">

    <ul id="navigation">
      <li><a href="index.html">Home</a></li>
      <li><a href="fruit/oranges.html">Oranges</a></li>
      <li><a href="fruit/strawberries.html">Strawberries</a></li>
      <li><a href="https://en.wikipedia.org/wiki/Blueberry">Blueberries</a></li>
    </ul>
  </div>
  
  <div id="content_container">
    <!-- Page content goes here -->
  </div>
</body>
</html>
CSS Loading Order Matters!

Styles are applied in the order they load. Later styles can override earlier ones.

  1. Reset CSS (removes browser defaults)
  2. A1.css (your base styles)
  3. Page-specific CSS (color overrides)

Common A1 Mistakes
  • Forgetting reset CSS - Must load the reset stylesheet first!
  • Wrong CSS order - Page-specific CSS must load AFTER A1.css
  • Missing viewport meta tag - Required for mobile responsiveness
  • Inline styles - All styles must be in CSS files
  • Wrong file paths - Be careful with ../ for parent folder
  • Wrong header height - Must be exactly 75px visible + 20px padding
  • Forgetting margin-top on body - Needs 100px to avoid header overlap
  • Not using flexbox for navigation - Required for positioning
